    Setting up the workspace

    To use the scss features you need to install the Prepros program (free). The program acts as a gulp build. Then the same folder with the template is opened in the code editor. In this case it is Visual Studio Code.

    Files and folders:

    This template used the scss preprocessor. The files in the dist folder can be used to correct the template. But you can also make changes from the #source folder. For this purpose it is necessary to use programs - assemblers. For this template we used the program Prepros (free).

    Screenshots from the Visual Studio Code program.
    All files have comments. The dist/index.html file is divided into sections with comments.

    When using scss, all HTML sections are in separate HTML files. The assembly and sequence of files is shown in the file #source/index.kit.
    You can easily change the sequence here, which will show up in the original index.html file

    All scss files are assembled by the builder into a common dist/css/style-dist.css file. The scss files are named after different themes. The sequence is displayed in the file #source/scss/main.scss. All js files are grouped in the same way (#source/js/main.js)

    The template does not use php or json files, because it is not a finished page, but only a template. It can be used, for example, as a wordpress template.
